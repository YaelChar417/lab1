SmartSnack:

SmartSnack es un sistema de alimentación para peces automatizado, diseñado para entornos domesticos, combinando una alimentación precisa, medición de temperatura y el monitoreo de una turbidez del agua para asegurar un entorno optimo para los peces, el cual cuenta con las siguientes características:

  • Dispensador automático de comida: Cuenta con una pieza diseñada en 3D y un servomotor para actuar como el dispensador de comida
  • Sensor de temperatura: Se utiliza el sensor de temperatura DS18B20 para monitorear la temperatura del agua
  • Sensor de turbidez: Se utiliza el sensor de turbidez DS18B20 el cual cuenta con la capacidad de ser sumergible en agua, bridando una mejor medición
  • Conectividad IoT: Los datos son enviados a través del wifi a una base de datos y desplegados en una pagina web para la vista del usuario, esto a través del microcontrolador ESP32
  • Interfaz web: Se diseño una pagina web donde aparecen tanto valores númericos como imagenes para facilitar la comprensión del usuario

Pregunta y respuesta

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    El internet se refiere a una red de computadoras conectadas entre sí, mientras que la World Wide Web o Web es una colección de páginas que funciona con internet. Con esto nos damos cuenta que la web depende del internet. Con una analogía el internet serían las carreteras y caminos y la web son los carros o personas que circulan por esos caminos

  • ¿Cuáles son las partes de un URL?

    Generalmente una URL esta conformada por 4 partes, un Esquema que define el tipo de protocolo que se usa para acceder a un recurso en internet, pueden ser HTTP o HTTPS, después está el nombre del host, que identifica quien tiene el recurso, continuando con la ruta o camino que identifica el recurso especifico que se desea acceder y finalmente puede contener una cadena de consulta que va al final de la URL, por lo general esta es una cadena de pares, con parametro y valor.

    Ejemplo de una URL: Protocolo://nombreHost:port/ruta?consulta.

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    Los metodos HTTP tienen la función de hacer distintas acciones en un sitio web, GET: se utiliza para recuperar datos, HEAD: sirve de la misma forma que GET pero no devuelve el cuerpo de mensaje en la respuesta, POST: se utiliza para enviar una entidad o recurso en especifico, PUT: funciona para reemplazar un recurso por otro en el destino, PATCH: sirve para aplicar modificaciones parciales a un recurso, finalmente DELETE: elimina un recurso en especifico.

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    Se deberá implementar un método POST debido a que este se encarga de enviar datos al destino esperado, creando un nuevo registro en una base de datos por ejemplo, ningún otro método tiene esta misma funcionalidad, solo se podría parecer PUT debido a que igual se envían datos pero no se debe usar este porque reemplaza a un registro. En conclusión POST es el indicado.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    El método encargado de hacer esta tarea es el método GET, debido a que esta se encarga de pedir y recuperar la información, en este caso con la URL se esta accediendo a una página web o cualquier otro recurso, devolviendo ese recurso.

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    Todo lo contario, deberías de agradecer ampliamente, un codigo 200 indica que el servidor ha recibido la petición y ha respondido correctamente a la petición

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    Si y no, porque este error que representa que el contenido no fue encontrado, esto se puede deber tanto a un error propio al momento de desplegar un proyecto, o que el cliente ingreso mal la URL o intenta acceder a una página que realmente no existe.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    Para un error 500, si es responsabilidad del desarrollador, ya que este error por lo general se refiere a un error del servidor, esto ocurre por motivos muy distintos, desde una mala consulta, la base de datos falló en algun punto, una mala conexión de la API, errores de sintaxis, etc. Por lo que si es responsabilidad del desarrollador arreglar esto.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    Desaprobado o despreciado, son formas de decirle a elementos de HTML que han sido reemplazados por estructuras nuevas, algunos ejemplos de etiquetas desaprobadas son acronym, usada para los acronimos, big, que aumentaba el tamaño de letra de los caracteres que se encontraran dentro de esta etiqueta, tt, que etiquetaba texto de fuente de espaciado fijo, incluso atributos como name , que se utilizaba para nombrar elementos, border, usado en tablas u otros elementos, para brindarle un borde a estos elementos, frame, con el mismo uso que border, pero este establece los lados del borde que eran visibles, rules, que establecia que bordes interiores de la tabla eran visibles.

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    La compatibilidad es una gran diferencia, debido a que en HTML5 la mayoría de sus funciones son compatibles con los navegadores, contando con nuevas etiquetas que son total o parcialmente compatibles en muchos navegadores.

    El soporte multimedia es también una gran diferencia, pudiendo agregar audios, videos, animaciones y juegos, cosa que antes requería de javascript pero ahora HTML5 cuenta con etiquetas que hacen muy simple este proceso

    HTML5 cuenta con la ventaja de ser mucho más rápido que sus versiones anteriores, esto gracias a que con HTML5 se han traido mejores normas para garantizar que los sitios web funcionen con mayor fluidez, admitiendo ahora el multithreading con javascript, haciendo que el procesador utilice más potencia.

    En cuanto al almacenamiento web HTML4 solo admite cookies, por lo que almacenar la información del usuario en una cookie que solo permite guardar 4 kb es muy dificil, mientras que en local la diferencia es muy considerable permitiendo guardar entre 5 y 10 mb, además que este no se borra al contrario que las cookies. HTML5 permitió el almacenamiento en local a través de una API de almacenamiento web que además permite almacenar en bases de datos web SQL.

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    Algunos componentes de una tabla son las td que son un conjunto de celdas, tr las filas que pueden estar organizadas en diferentes secciones dependiendo la ubicación de la tabla, como lo son thead, donde estará en la cabecera de la tabla, tbody o el cuerpo de la tabla y finalmente tfoot, que es el pie de tabla, asimismo puede tener la etiqueta caption que es un titulo que aparece en la parte superior de la tabla y finalmente col y colgroup que hacen referencia a una columna y grupo de columnas respectivamente.

    En cuanto a los estilos existen para estilizar los bordes con etiquetas como border, border-collapse, border-radius o border-style, se puede ajustar su width y height, se puede cambiar el color de fondo y de texto con background-color y color, se puede ocupar mayor espacio de una celda con colspan, etc.

  • ¿Cuáles son los principales controles de una forma HTML5?

    Los principales controles de forma en HTML 5 son los input, que funcionan para escribir texto de distintas variedades como nombres, fechas, cantidades, etc. Los button que tienen la función de hacer clic en ellos y desencadenar una o más acciones, select el cual crea un menu donde puede seleccionar varias opciones un usuario, esto ultimo es tambien otra, option que es el texto de una opcion dentro del select, finalmente se encuentra el textarea, que sirve para escribir una gran cantidad de texto.

  • ¿Cuál es el ciclo de vida de los sistemas de información?

    Consta de 7 fases: 1.-Planificación: se define el alcance del proyecto, justificarlo y preparar el diseño, tiene que ver con el analizar las necesidades del proyecto y la identificación de requisitos. 2.-Análisis: se busca concretar los requisitos del sistema, identificar los procesos y definir los requerimientos funcionales y no funcionales, etc. 3.-Diseño: en esta parte se determina como el sistema de información cumplirá con los requisitos aplicables, identificando soluciones potenciales y evaluarlas. 4.-Desarrollo: significa el inicio de la producción y se caracteriza por un constante cambio, consta en el desarrollo de los modulos del proyecto para cumplir con las funcionalidades. 5.-Integración: se corrobora que el diseño propuesto cumple con los requisitos de negocio establecidos y se empieza a probar el programa para verificar que todo funcione correctamente. 6.-Implementación: en esta parte se busca someter a prueba el sistema y la capacitación de los usuarios finales. 7.- Mantenimiento: consiste en ajustar cambios para mejorar el rendimiento o reducir los problemas que aparezcan.

  • ¿Cuál es el ciclo de desarrollo de sistemas de información?

    El desarrollo de sistemas en un proceso que consiste en dos etapas principales, que son el analisis de requerimientos y el diseño de sistemas, este inicia cuando la administración de la empresa decide que necesita un sistema de información debido a un cierto aspecto que se puede mejorar, y termina cuando el sistema se ha implementado exitosamente y cumpliendo con los objetivos planteados.

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    De acuerdo a una pequeña investigación realmente es muy util la aclaración "!important" ya que su uso se basa en sobreescribir la presedencia de los estilos, lo cual beneficia a usuarios que tengan necesidades especiales, sobreescribiendo la presedencia en cosas como colores o tamaños de fuente y también sirve para que si extensiones maliciosas quieren modificar esta presedencia, el valor de "!important" lo impide.

  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    Se debe escoger con cuidado una imagen ya que estas influyen mucho en los tiempos de carga de nuestra pagina web y en el SEO, además de ayudar a los bots de rastreo a dar reconocimiento de la web.

    Se debe tener en cuanta el formato de imagen ya que algunos son más pesados que otros y cada uno tiene una función para la cual es mejor, se recomienda usar palabras clave a la hora de nombrar estas imagenes y utilizar atributos como title que da una descripción de lo que se ve en la imagen o alt que ayuda cuando no se puede ver la imagen poner un texto alternativo.

  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    En cuanto al porcentaje se recomienda que este sea utilizado para definir margenes, relleno, espaciado o altura, en el caso de px y pt al ser estas unidades absoulutas se representarán iguales en cualquier dispositivo, por ello es mejor usar REM o EM que se adaptan mejor en diferentes dispositivos y sirven para asegurar una accesibilidad mayor.

  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    Al usar una versión minimizada del css se reduce el tamaño del archivo y esto hace que se cargue de forma más rapida y por ende también lo haga la pagina.
